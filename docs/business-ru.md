# Система управления реабилитационным центром — Бизнес-документация

## 1. Общее описание

**Название продукта:** Rehab Center  
**Тип:** Внутреннее веб-приложение (PWA — устанавливается на телефон как мобильное приложение)  
**Назначение:** Цифровая платформа для управления жилыми реабилитационными центрами

Система объединяет все ежедневные операции реабилитационного учреждения в одном инструменте: учёт пациентов, управление медикаментами, смены консультантов, финансы, сессии терапевта и многое другое. Разработана для использования персоналом на мобильных устройствах во время смен.

Система поддерживает **многодомовые организации** — одна организация может управлять несколькими реабилитационными домами из одного аккаунта с разграничением доступа по сотрудникам.

## 2. Роли пользователей

### org_manager — Менеджер организации
- Доступ ко **всем домам** организации
- Может создавать и управлять всеми сотрудниками
- Полный доступ ко всем модулям
- Используется директором или администратором организации

### manager — Менеджер дома
- Доступ к **своему дому** (или нескольким, если явно предоставлен)
- Управляет расписанием персонала
- Подтверждает или отклоняет последствия и отсутствия
- Полный операционный доступ внутри своего дома

### counselor — Консультант
- Доступ к **своему дому**
- Управляет ежедневными операциями с пациентами
- Фиксирует смены, пишет ежедневные итоги
- Работает с телефонами, отсутствиями, последствиями
- Не может управлять медикаментами (только чтение)

### doctor — Врач
- Доступ к **своему дому**
- Полное управление медикаментами (добавление, редактирование, удаление, выдача)
- Может просматривать всю информацию о пациентах

### therapist — Психотерапевт
- Доступ к **своему дому**
- Управляет терапевтическими сессиями и привязкой пациентов
- Может писать заметки к сессиям и отмечать срочные случаи

## 3. Модули системы

### 3.1 Панель управления (Dashboard)
Главный экран после входа в систему. Отображает актуальное состояние дома в реальном времени:

- Общее количество активных пациентов, пациентов в отсутствии
- Индикатор настроения (средний показатель по всем пациентам)
- Алерты — пациенты, требующие особого внимания
- Ближайшие смены
- Последствия, ожидающие подтверждения
- Срочные пометки от терапевтов
- Сводка по телефонам (сколько телефонов выдано)
- Групповые занятия, запланированные на сегодня

### 3.2 Пациенты (Patients)
Основной модуль. Отображает всех активных пациентов текущего дома.

**Возможности:**
- Просмотр списка активных пациентов (сортировка по дате поступления)
- Просмотр архивированных (выписанных) пациентов
- Добавление нового пациента (имя, дата рождения, дата поступления, комната)
- Открытие профиля пациента
- Выписка пациента с указанием типа (успешная, самовыписка, побег)

**Профиль пациента содержит:**
- Основная информация: имя, дата рождения, дней в реабилитации, текущая комната, оценка настроения
- Флаг алерта (отмечает пациента для срочного внимания)
- Вкладка «Медикаменты» — список текущих препаратов
- Вкладка «Отсутствия» — история домашних визитов и выходов
- Вкладка «Финансы» — личный баланс и транзакции пациента
- Вкладка «Терапия» — назначенный терапевт и история сессий
- Вкладка «Последствия» — дисциплинарные записи

### 3.3 Комнаты (Rooms)
Управление физическими комнатами дома.

**Возможности:**
- Просмотр всех комнат сгруппированных по корпусам с текущей заполненностью
- Добавление новых комнат (номер, корпус, вместимость)
- Редактирование данных комнаты
- Удаление пустых комнат
- Размещение пациентов без комнаты в комнату
- Перевод пациента из одной комнаты в другую
- Освобождение комнаты (убрать пациента из комнаты)

### 3.4 Медикаменты (Medications)
Два подмодуля:

**Менеджер медикаментов (Med Manager):**
- Доступен только врачу
- Добавление, редактирование, удаление медикаментов для пациентов
- Настройка расписания приёма (утро / день / вечер / ночь)
- Указание дат начала/окончания курса и назначившего врача

**Выдача медикаментов (Medication Distribution):**
- Ежедневный чеклист выдачи медикаментов
- Отметка о выдаче дозы с временем и именем сотрудника
- Просмотр невыданных доз для текущего временного слота

### 3.5 Группы (Groups)
Учёт групповых терапевтических занятий.

**Возможности:**
- Просмотр всех групповых занятий текущего дома
- Создание нового занятия (дата, тема, ведущий)
- Отметка посещаемости пациентов (присутствовал / отсутствовал)
- Добавление заметок к занятию

### 3.6 Телефоны (Phones)
Учёт временного использования телефонов пациентами (телефоны обычно ограничены в реабилитации).

**Возможности:**
- Выдача телефона пациенту с временем возврата
- Отметка о возврате телефона
- Автоматическая пометка опозданий с возвратом
- Просмотр истории использования телефонов

### 3.7 Смены (Shifts)
Фиксация передачи дежурств консультантов.

**Возможности:**
- Просмотр смен текущего дома по дате
- Создание записи о смене
- Принятие смены (подтверждение начала)
- Запись передачи: от кого принял и кому передал
- Добавление заметок к смене
- Завершение смены

**Статусы смены:** ожидает → АКТИВНА → завершена

### 3.8 Расписание (Schedule)
Визуальное расписание консультантов (кто работает в какой день).

**Возможности:**
- Просмотр календаря назначений консультантов на месяц
- Добавление консультанта на дату/тип смены
- Удаление записи из расписания
- Используется менеджерами для планирования покрытия

### 3.9 Последствия (Consequences)
Система дисциплинарных записей за нарушения правил пациентами.

**Возможности:**
- Создание записи о последствии (тип, описание, дата)
- Типы: устное предупреждение, письменное предупреждение, изъятие телефона, ограничение комнатой, обязательная встреча, другое
- Менеджер подтверждает или отклоняет каждое последствие
- Ожидающие подтверждения последствия отображаются на панели управления
- Полная история видна в профиле пациента

### 3.10 Финансы (Finance)
Два подмодуля:

**Финансы пациента:**
- У каждого пациента есть личный баланс (карманные деньги)
- Фиксация пополнений (семья отправила деньги) и расходов (покупки пациента)
- История операций с текущим балансом

**Касса дома (Cashbox):**
- Учёт наличных денег дома (доходы и расходы)
- Категории: расходные материалы, продукты, транспорт и т.д.
- Фиксация пересчётов кассы (физический подсчёт vs. ожидаемый остаток)
- Отметка расхождений

### 3.11 Отсутствия (Absences)
Управление разрешёнными выходами из учреждения.

**Возможности:**
- Запрос отсутствия для пациента (тип: домашний визит, поручения, медицинский, другое)
- Указание даты начала и ожидаемого возвращения
- Менеджер подтверждает или отклоняет запрос
- Отметка о возвращении пациента
- История отсутствий сохраняется по каждому пациенту

**Статусы отсутствия:** ожидает → подтверждено → активно → вернулся

### 3.12 Центр управления (Management Center)
Административный модуль, доступен преимущественно менеджерам и менеджерам организации.

**Вкладки:**
- **Персонал** — просмотр, добавление, редактирование сотрудников. Установка роли, телефона, доступа к домам. Новые пользователи получают пароль по умолчанию `1234`.
- **Расписание** — управление расписанием смен консультантов
- **Терапевты** — назначение терапевтов пациентам

### 3.13 Терапия (Therapy)
Управление терапевтическими сессиями, преимущественно для терапевтов.

**Возможности:**
- Просмотр назначенных пациентов
- Создание записей о сессиях (дата, тема, заметки)
- Отметка срочности сессии (Обычная / Срочная)
- Срочные сессии отображаются на панели управления и уведомляют дежурного консультанта
- Консультанты могут добавлять ответные заметки к сессиям

### 3.14 Ежедневный итог (Daily Summary)
Отчёт консультанта по окончании смены.

**Возможности:**
- Написание общего итога дня
- Индивидуальные заметки по каждому пациенту
- Отправка итога (с временной меткой)
- Менеджеры могут просматривать все поданные итоги по дате

## 5. Логика многодомовой системы

- Каждый пользователь принадлежит к одному основному дому (`houseId`) или имеет доступ к нескольким домам через `allowedHouses`
- Пользователи с `allHousesAccess: true` (обычно org_manager) видят все дома и могут переключаться между ними
- При переключении дома все данные (пациенты, комнаты, смены и т.д.) перезагружаются для выбранного дома
- Пациенты, комнаты и все операционные данные всегда привязаны к конкретному дому

---

## 6. Ключевые бизнес-правила

1. Пациент может находиться только в одной комнате одновременно
2. Комнату нельзя удалить, если в ней есть пациенты
3. Пациента нельзя выписать без указания типа выписки
4. Последствия требуют подтверждения менеджера
5. Отсутствия требуют подтверждения менеджера до выхода пациента
6. Записи о выдаче медикаментов формируются ежедневно по расписанию приёма каждого пациента
7. Терапевтические сессии с пометкой «Срочно» отображаются на панели управления
8. Смена консультанта должна быть явно принята, прежде чем станет АКТИВНОЙ
9. Время возврата телефона отслеживается; опоздания помечаются автоматически
10. У каждого пациента может быть назначен только один терапевт (или ни одного)

## 8. Объём MVP

В MVP (версия 1) включено:

- ✅ Авторизация (вход, JWT-сессия, автовосстановление при перезагрузке)
- ✅ Поддержка нескольких домов с переключателем домов
- ✅ Управление пациентами (создание, просмотр, редактирование, выписка)
- ✅ Управление комнатами (создание, редактирование, удаление, размещение пациентов)
- ✅ Управление пользователями (создание и редактирование аккаунтов сотрудников)
- ✅ Управление медикаментами (добавление, редактирование, удаление, чеклист выдачи)
- ✅ Групповые занятия с учётом посещаемости
- ✅ Учёт телефонов (выдача, возврат, пометка опоздания)
- ✅ Смены консультантов (создание, принятие, передача, завершение)
- ✅ Планирование расписания смен
- ✅ Последствия (создание, подтверждение/отклонение)
- ✅ Финансы пациентов и касса дома
- ✅ Управление отсутствиями (запрос, подтверждение, возвращение)
- ✅ Терапевтические сессии и назначение терапевтов
- ✅ Ежедневные итоги смены
- ✅ Панель управления с актуальным обзором дома
- ✅ PWA — устанавливается на iOS/Android, работает офлайн для закэшированных экранов

**Не входит в MVP:**
- Push-уведомления
- Загрузка документов и файлов
- Отчёты и экспорт аналитики
- Внешние интеграции (аптеки, страховые компании и т.д.)
- Журнал аудита действий пользователей
